# Changelog

See [PLAN.md](PLAN.md) for the canonical feature-by-version table; this file mirrors it in release-please format. Entries for v0.9 through v1.4 were backfilled manually — compare URLs will be regenerated by release-please on the next tagged release.

## [1.4.0] (2026-04-19)

### Features

* Frozen `State` + `ForkContext` (new `termiclaw/state.py`); all state transitions go through `dataclasses.replace` or `with_*` helpers
* Pure `decide` core (`termiclaw/decide.py`) with exhaustive match over the `Event` sum type
* `apply(cmd, ports)` imperative shell (`termiclaw/shell.py`) with exhaustive match over the `Command` sum type
* `Ports` Protocol bundle (`termiclaw/ports.py`) + default impls (`termiclaw/runtime.py`); tests use `tests/unit/fakes/`
* `Transition` product type, `LoopTick(summarize_ready, session_alive)` synthetic event
* Pre-commit guards: `no-monkeypatch`, `exhaustive-match`
* Stall detection rewritten pure: `detect_stall` returns `tuple[StallState, StallSignal]`

## [1.3.0] (2026-04-19)

### Features

* MCTS real forks — `_parent_state_for_fork` loads parent run from SQLite, child gets `--fork-session` on first planner call
* `termiclaw mcts-show <search-id>` ASCII tree view
* `termiclaw export <run-id> --format atif` — ATIF v1.6 export
* `termiclaw tag` + `termiclaw failures` — failure categorization (closed `FailureCategory` enum)
* Async summarization (fixes BUG-29): `SummarizationWorker` runs the 3-subagent pipeline off-path
* Pre-commit `no-object-signatures` guard

## [1.2.0] (2026-04-12)

### Features

* Task verifier (`VerifierSpec`, bash exit-code check in a `[verifier]` TOML section)
* `termiclaw eval <dir>` — batch-run a directory of TOML tasks with pass/fail reporting
* MCTS scaffolding (`MctsSearch`, UCB1, parallel playouts) — fork path mocked

## [1.1.0] (2026-04-10)

### Features

* `Result[T, E]` type (`Ok[T] | Err[E]`) via PEP 695 generics; callers pattern-match
* Custom exception hierarchy (`TermiclawError` + subclasses); no bare `Exception`
* Validator combinators (`termiclaw/validate.py`) narrow `dict[str, object]` at the JSON boundary

## [1.0.0] (2026-04-09)

**Breaking release.**

### Features

* Docker-only execution. `termiclaw/tmux.py` removed. Each run provisions a fresh container (`ubuntu:24.04` base with tmux, content-hashed image tag).
* Modes purge: dropped `ParsedCommand.mode`, `block_until_idle`, `environment`; no more user-facing mode knobs
* `Config` stabilized; subsequent changes are additive

## [0.12.0] (2026-04-08)

### Features

* Session forking via `--fork-session` — `termiclaw fork <run-id>` seeds a fresh Claude Code session from state-dump artifacts

## [0.11.0] (2026-04-07)

### Features

* Blocking command wait (initial): `; tmux wait -S done` wrapper around commands
* State-dump artifacts: per-run `WHAT_WE_DID.md`, `STATUS.md`, `DO_NEXT.md`, `PLAN.md` regenerated at interval/token triggers

## [0.10.0] (2026-04-06)

### Features

* Agent must self-unblock: stall detection (hashed screen + hashed keystroke streak) → nudge → force-interrupt (`C-c`) escalation
* Full scrollback capture: `capture-pane -S -` (no line cap)
* Shell-quoted keystrokes (initial, later removed in v1.0)

## [0.9.0] (2026-04-05)

### Features

* `Config.max_command_length` wired to `send_keys` (was a dead parameter)
* Token-based context accounting: `planner.extract_usage` → per-call input/output/cache tokens on `RunState`; chars become secondary
* `--json-schema` structured output — Claude-side validation of the envelope
